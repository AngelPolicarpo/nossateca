# 00 - Feature Template

Use este formato para documentar qualquer nova feature.

## Contexto
- Problema que a feature resolve.
- Limite de escopo do ciclo atual.

## Responsabilidades
- Arquivos principais no frontend.
- Comandos backend envolvidos.
- Modelos de entrada/saida impactados.

## Fluxo funcional
1. Acao do usuario.
2. Comando invocado.
3. Persistencia/evento retornado.
4. Atualizacao de UI.

## Regras de negocio
- Validacoes obrigatorias.
- Estados permitidos.
- Mensagens de erro esperadas.

## Decisoes de arquitetura
- Por que o fluxo foi implementado dessa forma.
- Trade-offs aceitos.

## Testes minimos
- Caso feliz.
- Erro principal.
- Regressao de contrato.

## ⚠️ Inconsistências encontradas
- Preencher quando houver divergencia entre docs, codigo e comportamento observado.
